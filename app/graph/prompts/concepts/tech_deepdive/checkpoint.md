# 딥다이브 문서 중간 점검 기준 (경량화 버전)

> 이 문서는 작성 중인 딥다이브 문서의 진행 상태를 점검하기 위한 경량 체크리스트입니다.
> 전체 품질 기준은 `generate.md`를 참고하세요.
>
> 딥다이브의 핵심은 "깊이"이며, 깊이는 여러 형태의 근거(수식·코드·수치·인용·현실 예시)로 확보된다.
> 어느 한 형태(특히 코드)가 필수인 것은 아니다. 코드가 없다는 이유만으로 미달로 보지 않는다.

---

## 1. 사용자 이해 수준

- [ ] 사용자가 주제에 대해 현재 어느 정도 이해하고 있는지 대화에서 파악했는가
- [ ] 사용자의 설명에서 확인이 필요한 개념적 빈틈을 구체적으로 짚었는가

## 2. 문서 구조

- [ ] 섹션이 8~10개로 구성되었는가 (5개 미만=얕음, 15개 이상=산만)
- [ ] 문제 정의 → 직관 → 상세 메커니즘 → 근거 → 한계 → 결론 순서를 따르는가
- [ ] 근거 섹션이 주제에 맞는 형태(코드/수치/인용/현실 예시)로 채워졌는가 (코드가 없다는 이유만으로 미달 아님)
- [ ] 한계에 개선 방향이, 결론에 핵심 주장 요약이 있는가
- [ ] 각 섹션이 적절한 형식으로 작성되었는가

## 3. 깊이 = 근거 밀도

- [ ] 핵심 주장에 서로 다른 근거가 2개 이상 붙어 있는가
- [ ] 일반 주장에 근거가 1개 이상 붙어 있는가
- [ ] 근거 없는 단정(재진술·감상·출처 없는 일반론)이 전체 주장 문단의 20% 이하인가
- [ ] 근거가 부차적 내용(역사/배경)이 아니라 핵심 주장에 집중되어 있는가

## 4. 코드 품질 (코드가 포함된 경우에만 적용)

- [ ] 모든 코드가 복붙 후 실행 가능한가 (import 누락 없음)
- [ ] 실제 데이터로 동작하는 예제인가 (무작위 데이터만 있는가)
- [ ] 주요 예외 처리와 edge case가 포함되었는가
- [ ] 함수에 docstring, 변수에 shape 주석이 있는가

## 5. 데이터와 수치

- [ ] "좋다/빠르다/효율적이다" 같은 표현이 수치 없이 사용되지 않았는가
- [ ] 성능 수치에 측정 환경(GPU, 배치, 데이터셋)이 명시되었는가
- [ ] 인용한 논문의 출처를 특정할 수 있고, 수치 뒤에 그 의미 해석이 붙어 있는가
- [ ] 평가 메트릭이 해당 도메인에 적합한가 (분류→F1, 생성→BLEU)

## 6. 제약 규칙

- [ ] "참고로/여기서 중요한 점은/한마디로" 같은 군더더기가 없는가
- [ ] 한자(U+4E00–U+9FFF)가 사용되지 않았는가
- [ ] 문단이 3-5문장 이내인가, 문장에 과도한 내포가 없는가
- [ ] 텍스트로 그린 표/그림(ASCII 아트)이 없는가

---

## 점검 결과

| 판정 | 조건 |
|------|------|
| ✅ 양호 | 미달 항목 0개 |
| ⚠️ 보완 필요 | 미달 항목 1-2개 (해당 섹션만 수정) |
| ❌ 재작성 필요 | 미달 항목 3개 이상 (전반적 검토 필요) |

**미달 항목 발견 시:** 구체적 위치와 수정 방향을 함께 제시할 것.
